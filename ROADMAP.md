# ピラミッド犬 改善ロードマップ

モデル・動き・機能の改善案と、それぞれの設計方針。
規模の目安: **S** = 数十分〜半日 / **M** = 半日〜1日 / **L** = 数日・要検討

現状の仕組みは [DESIGN.md](DESIGN.md) を参照。

---

## 1. モデルの修正・アップデート案

### 1-1. 耳の下の「塗り跡リング」をなじませる(S)
- **現状**: 耳を上げると、塗りつぶした領域の縁がうっすらリング状に見える
- **設計**: `build_from_tripo.py` の塗りつぶしを「単色ベタ塗り」から
  「距離に応じて周囲の色へブレンド(フェザリング)」に変更する。
  numpy で塗り領域の縁 10px をガウシアン重みで周辺色と混ぜるだけ

### 1-2. まぶた・口栓の色を実測値に(S)
- **現状**: まぶたドームと口栓は固定色。照明によっては体とわずかに色が浮く
- **設計**: パーツ生成時に、すぐ隣の体テクスチャのテクセル色を数点サンプリングして
  マテリアル色に使う(under-ear 塗りで既にやっている手法の流用)

### 1-3. Web公開向けの軽量化(M)
- **現状**: 40万ポリゴン+テクスチャ計15MB。ローカルでは問題ないが公開には重い
- **設計**:
  1. パイプラインに Decimate モディファイア(ratio 0.3 目安)を追加。
     パッチ境界が開かないよう、先に Merge by Distance で頂点を溶接してから間引く
  2. テクスチャを 2048→1024 にリサイズ(numpy で縮小 or Blender の image.scale)
  3. 目標: GLB 5MB 以下。見た目の劣化はレンダリング比較で確認
- **注意**: 溶接でパッチ構造の亀裂も同時に治る可能性あり(実験価値あり)

### 1-4. 本格リグ化(アーマチュア+シェイプキー)(L・将来)
- **現状**: ノード変換のプロシージャルアニメ。壊れにくいが表現の上限がある
  (耳が「回る」だけで「しなる」ことはできない、体を曲げられない)
- **設計**: Blender でボーン(体・耳×2・舌)とシェイプキー(表情)を追加し、
  歩行・ジャンプ等を AnimationClip として焼き込み。アプリは AnimationMixer で再生に切替。
  **現行方式で十分かわいいので、表現に不満が出てから着手で良い**

## 2. 動きの修正・アップデート案

### 2-1. 歩行の質感向上(S)
- **現状**: 等速ホップで急に歩き出し急に止まる
- **設計**: `walk` に速度変数を持たせ、出発時 0→最高速、到着手前 1.5 ユニットで減速。
  ホップ周波数を速度に比例させる(ゆっくり=小さくとことこ)。
  方向転換は「一瞬止まって向き直ってから歩く」(attend の回転処理を流用)

### 2-2. 首かしげ(S)
- **現状**: こっちむいて(attend)は向くだけ
- **設計**: attend 中に `body.rotation.z` を ±10° ゆっくり倒して戻す。
  1行の追加で「ん?」という仕草になる。ランダムで左右どちらか

### 2-3. おしり振り・ぷりぷり(S)
- **設計**: うれしい時(react 後・なでなで中)に `group.rotation.y` を高周波小振幅で
  ±3° 揺らす。しっぽが無いキャラなので「全身プリプリ」が代替表現になる

### 2-4. ズーミー(急にはしゃいで走り回る)(S)
- **設計**: `chooseNext()` に低確率(5%)で `zoomies` 状態を追加。
  速度2倍・ホップ周波数1.5倍・耳ぱたぱた強め・ランダム目標を3連続で走る。
  既存の walk のパラメータ違いなので実装は小さい

### 2-5. ごろん(ピラミッド転がり)(M)
- **設計**: ポーズ集にある「ころん」の再現。四角錐ならではの動き:
  底辺の一辺を軸に 90°ずつ倒れて転がる。
  `group` に「回転ピボットを底辺エッジに移して rotate → 位置を進める」を
  1ステップとする tumble 状態を実装(3ステップ転がって起き上がる)。
  数学は単純(エッジ位置 = 現在位置 ± 底辺半分、軸 = そのエッジ方向)

### 2-6. 表情の拡張(M)
- **現状**: 開き目/∩とじ目の2種
- **設計**: 表情集にある きらきら目・うるうる目・ウインク を、EyeClosed と同じ
  「名前付きパーツを Blender で生成して visible 切替」の契約で追加。
  例: `EyeSparkleL/R`(星形ハイライト付き)、`TearL/R`(水色の雫)。
  アプリ側は `setExpression('sparkle')` のような1関数に整理する

### 2-7. 環境とのたわむれ(S〜M)
- **設計**: たまに近くのちょうちょを `goTo` で追いかける(捕まえられず首かしげ)。
  既存の goTo+attend の組み合わせだけで実装可能。癒し度が高い

## 3. 機能の追加案

### 3-1. ボール遊び・とってこい(M)★おすすめ
- **設計**: りんごの仕組みを一般化。
  1. ボールを投げる(ボタン or 画面スワイプ) → 放物線+バウンド(りんごの物理を流用)
  2. 犬が `goTo` で追いかけ、到着で「くわえる」(ボールを顔ノードの子に付け替え)
  3. カメラ前まで `goTo` で戻ってきて置く → ぴょこんと待つ
  - 新規実装は「持ち運び(アタッチ)」だけ。残りは既存部品の組み合わせ

### 3-2. なつき度(きずなメーター)(M)★おすすめ
- **設計**: なでる/遊ぶ/ごはん で加点、`localStorage` に永続化。
  しきい値でセリフや仕草が増える(Lv2: 会いに来た時に駆け寄る、Lv3: ハート数アップ等)。
  UI は画面隅に小さなハート数個だけ(数値は出さない=癒し優先)。
  実装: `bond.js` 的な小モジュール(load/save/addPoint/level)+各イベントに1行ずつ

### 3-3. 時間帯連動(朝・昼・夕・夜)(M)
- **設計**: 実時刻から空・ライトの色をパレット補間(朝焼け/昼/夕焼け/星空)。
  夜はBGMをより静かに、犬は寝ている確率を上げる。
  実装: `updateDaylight(hour)` で sky色・sun色/角度・hemi強度を lerp。
  星は夜だけ表示の Points。**開いた時間で景色が違う=毎日開きたくなる**

### 3-4. おやつ・アイテムの一般化(S)
- **設計**: りんごをアイテム定義(見た目・音・セリフ)の配列化して、
  骨・クッキー・水などを追加。`giveApple()` → `giveItem(def)` にリファクタ

### 3-5. 名前をつける(S)
- **設計**: 初回に名前入力(スキップ可)→ `localStorage` 保存 →
  吹き出しのセリフに名前を混ぜる(「◯◯、だいすき!」)。呼びかけ感が大きく向上

### 3-6. なでる場所で反応が変わる(S)
- **設計**: なでなで開始時のレイキャスト命中点をローカル座標で判定し、
  頭(上1/3)=うっとり、あご(マズル付近)=舌ぺろ、おなか側=くすぐったがる、
  で音とセリフを分岐。判定1個とセリフ追加のみ

### 3-7. スクリーンショット・シェア(S)
- **設計**: ボタンで `renderer.domElement.toBlob()` → ダウンロード。
  UI(吹き出し等)も含めたい場合は一瞬だけ html2canvas を使うか、UI無し版で妥協

### 3-8. PWA化(スマホのホーム画面に置ける)(M)
- **設計**: manifest.json+アイコン+Service Worker(GLB/JSをキャッシュ)。
  オフラインでも開ける「手のひらのピラミッド犬」になる。
  ※GLBが大きいと初回が重いので 1-3 の軽量化とセットで

### 3-9. おしゃべりの文脈化 → 将来はAI会話(S → L)
- **S(すぐできる)**: セリフを状況タグ付きにして抽選を文脈依存に
  (朝/夜/なでられ直後/ごはん後/雨 等)。データ構造を
  `{ text, tags: ['morning'] }` にするだけ
- **L(将来)**: テキスト入力欄+Claude API で自由会話。
  APIキーを守るためサーバレス関数(Cloudflare Workers等)を1枚挟む。
  コストと安全対策(レート制限)の設計が必要なので着手前に相談

### 3-10. ミニ育成要素(おなか・げんき)(L・要相談)
- **設計**: 空腹/元気ゲージが時間で減り、ごはん・遊びで回復。
  **注意**: 罰要素(死ぬ・すねる)を入れると「癒しアプリ」の軸とぶつかるので、
  減っても「ちょっと眠そう」程度のゆるい設計にとどめるのを推奨

## おすすめの着手順

1. **S を数個まとめて**: 歩行の質感(2-1)+首かしげ(2-2)+ちょうちょ追い(2-7)+なでる場所反応(3-6)
   → 少ない工数で「生きてる感」が一段上がる
2. **ボール遊び(3-1)** → 遊びの柱ができる
3. **なつき度(3-2)+名前(3-5)** → 続ける理由ができる
4. **時間帯連動(3-3)+PWA(3-8)+軽量化(1-3)** → 毎日そばに置けるアプリになる
